# binproto-demo

Working demo of a verified binary protocol.

## Run

```bash
# Basic demo
./demo.sh

# With custom message
./demo.sh "Your text here"

# View database (stub)
./binproto-view test.bpdb

# Verify packet (stub)
./guardian packet.bin
What it shows
• Encrypt → Frame → Send → Receive → Decrypt pipeline
• Binary protocol, handwritten parser
• Dual-contour architecture (main + quarantine)
• Verified core: 152 theorems (Lean4/Coq/Agda)
Status
🔬 Demo only. Not for production.
🔐 Crypto: stub (XOR), not real AES.

## 🔐 Почему «чёрный ящик»?

Этот репозиторий — не библиотека и не фреймворк.  
Это **демонстрация подхода**.

> «Не верь языку — верь консенсусу».

Здесь консенсус достигается не через открытость кода,  
а через **результат, который можно запустить и проверить**.

### Что показано:
• Пайплайн работает: `encrypt → frame → send → receive → decrypt` ✅  
• Бинарный формат: рукописный парсер, контроль над каждым байтом ✅  
• Верификация: 152 теоремы в трёх системах (Lean4/Coq/Agda) как математический консенсус ✅  
• Архитектура: два контура данных (основной + карантин) — ошибка в одном не ломает систему ✅  

### Что не показано:
• Исходники крипто-слоя и фрейминга — чтобы не превращать демо в учебник  
• Детали инвариантов и доказательств — они живут в отдельном исследовательском контуре  
• Внутренний формат `.bpdb` — это часть «консенсуса через изоляцию»  

### Для кого это:
• Если ты инженер и хочешь увидеть, *что возможно* — запуски, смотри вывод, делай выводы.  
• Если ты исследователь и хочешь обсудить подход — пиши, поговорим.  
• Если ты ищешь готовое решение для продакшена — это не оно (пока).  

### Философия в двух словах:
Современная разработка часто говорит: «доверься абстракции, фреймворку, генератору».  
Этот проект говорит: «доверься доказательству, инварианту, консенсусу через верификацию».  

Не потому что «так красивее».  
А потому что в критических системах **язык может солгать, а консенсус — нет**.
